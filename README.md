# VPN Telegram-бот (Naive + Hysteria2, оплата через CryptoBot)

Стартовый каркас бота для автоматической продажи ключей к панели
[Panel-Naive-Hy2-by-RIXXX](https://github.com/cwash797-cmd/Panel---Naive-Hy2---by---RIXXX).
Стек: Python + aiogram 3 + SQLite + Crypto Pay API (@CryptoBot).

## Что делает бот

- Меню выбора протокола (Hysteria2 / NaiveProxy) и срока (тариф)
- Создаёт счёт в CryptoBot, ждёт оплату (автоматически опрашивает раз в 30 сек
  + кнопка ручной проверки)
- После оплаты сам создаёт пользователя на панели и присылает ссылку
  подключения + QR-код
- Хранит выданные ключи в SQLite, умеет продлевать их (новый счёт →
  пересчёт срока на панели → новая дата истечения)
- Раз в 12 часов напоминает о ключах, которые истекают в течение 2 дней

## ⚠️ Перед первым запуском — обязательно

### 1. Эндпоинты панели подтверждены

`panel_client.py` написан по реальному `server/index.js` твоей установки —
пути, поля и формат ответов проверены, не предположение. Коротко, что важно знать:

- Все ручки отвечают **HTTP 200 даже при отказе** — отказ это
  `{success: false, message: "..."}` в теле. `PanelClient` сам ловит это и
  поднимает понятную ошибку.
- Создание пользователя (`POST /api/naive/users` / `POST /api/hy2/users`)
  возвращает **готовую ссылку подключения** (`link`) — бот использует её
  как есть, не собирает сам.
- Продление (`PATCH .../users/:username`) — это **не** «добавить N дней к
  текущему сроку», а «установить срок = сейчас + N дней». Бот сам считает
  нужное число дней от текущего момента, чтобы получить желаемую итоговую
  дату, и сохраняет дату, которую вернула панель в ответ (`expiresAt`) —
  она авторитетная.
- Логин — обычная cookie-сессия (`rixxx_sid`), без отдельного API-токена.
  Бот логинится один раз и переиспользует cookie, перелогиниваясь
  автоматически при 401.

Если в будущем обновишь панель через `update.sh` и формат API поменяется —
обновлять нужно только `panel_client.py`.

### 2. Получи токен CryptoBot

Открой [@CryptoBot](https://t.me/CryptoBot) → Crypto Pay → Create App →
скопируй токен в `.env` (`CRYPTO_PAY_TOKEN`). Для тестов без реальных денег
есть @CryptoTestnetBot с тем же API.

### 3. Закрой панель от внешнего мира

Если порт `3000` (или `8080`) сейчас открыт наружу — заведи firewall-правило,
чтобы панель была доступна только с `127.0.0.1` (бот стучится локально) или
через SSH-туннель/VPN для твоего собственного доступа. Не оставляй панель
с дефолтным `admin`/`admin` смотрящей в интернет.

## Установка

```bash
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt

cp .env .env
# впиши BOT_TOKEN, PANEL_PASSWORD, CRYPTO_PAY_TOKEN, VPN_DOMAIN
```

Под твою установку: `PANEL_URL=http://127.0.0.1:3000` (SSH-only режим, бот
работает прямо на сервере `31.42.120.109`), `VPN_DOMAIN=cerberusvless.top`,
`PANEL_LOGIN=admin` — и обязательно смени `PANEL_PASSWORD` с дефолтного
`admin`, если ещё не сменил в настройках панели.

## Запуск

```bash
python3 bot.py
```

Для постоянной работы — через `pm2`, `systemd` или `screen`/`tmux`, например:

```bash
pm2 start bot.py --interpreter python3 --name vpn-bot
```

## Структура проекта

```
config.py        — переменные окружения, тарифы, список протоколов
database.py      — SQLite: платежи и выданные ключи
panel_client.py  — клиент API панели (эндпоинты подтверждены по коду панели)
cryptopay.py     — клиент Crypto Pay API (CryptoBot)
key_format.py    — сборка QR-кода по ссылке, которую возвращает панель
bot.py           — хендлеры aiogram, фоновые задачи, точка входа
```

## Известные ограничения этого каркаса (что стоит доделать перед продакшеном)

- Нет защиты от создания нескольких неоплаченных счетов одним юзером подряд
- Нет админ-команд (статистика, ручная выдача/отзыв ключа, список должников)
- Нет лимита пользователей на один VPS — учитывай нагрузку по трафику/CPU
  при росте базы
- SQLite ок для старта; при заметном потоке заказов — переезд на Postgres
- Опрос CryptoBot идёт polling-ом раз в 30 сек; при желании можно перейти на
  вебхуки Crypto Pay API для мгновенной выдачи
- Стоит свериться с условиями использования своего VPS-провайдера и
  локальным законодательством насчёт перепродажи доступа к VPN — это зона
  твоей ответственности как владельца сервиса
